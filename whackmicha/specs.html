1 Introduction

1.1 Purpose

The purpose of this page is to provide an overall description and understanding of the CISC 3140 project,
Whack-a-Professor. This document will list the specifications of the project such as game features, baseline requirements, and graphics.

1.2 Product Scope

The Whack-a-Professor software is a game designed in JavaScript, CSS and html for the CISC 3140 course project. The game page consists of a countdown timer of 20 seconds, a score which accumulates in increments of 10 points. A play button to start the game and 6 individual desks where the professors pop up. The objective of the game is to hit the professors with a mallet in order to accumulate points.

1.3 Intended Audience and Document Overview

Whack-a-Prof is intended for clients who would like a enjoy an online interactive game and for Professor Murray Gross to evaluate the work of class 3140-ET6 Group 1, which consists of a backbone team, a graphics team, a quality assurance team and a specifications team.

1.4 Defintions, Acronyms and Abbreviations

  • JavaScript - an object-oriented computer programming language commonly used to create interactive effects within web browsers.
  • CSS (Cascading Style Sheets) - is a computer language for laying out and structuring web pages.
  • HTML (Hypertext Markup Language) - a standardized system for tagging text files to achieve font, color, graphic,
    and hyperlink effects on World Wide Web pages.
  • End users - users whom the intended product is designed for.

2 Overall Description

2.1 Product Overview

Whack-a-Prof is a standalone software that is a variation on the traditional Whack-a-Mole game. The game is intended to run on any web browser that supports Javascript. The game is in a 3 by 3 grid arrangement where the Professor will pop up on the screens in which the user will then have to click on the Professor to “whack” him. The Player has only 20 seconds to get as high of a score as possible with the mallet which the Player controls with the mouse. The Professor pops up randomly and the score counter increases by 10 points when the Professor is hit.

2.2 Product Functionality

  • End users are users whom the intended product is designed for.
  • Professors pop up at random intervals
  • User has 20 seconds to get as much points as possible
  • User must use mouse cursor to “hit” the Professors
  • User will gain 10 points per successful hit.
  • There are sound effects to let the end user know whether the hit was successful or not.

2.3 Design and Implementation Constraints

The Whack-a-Prof is designed to run in any PC web browser, The game is not designed to work properly on mobile devices such as cellphones and tablets. The game has been restricted to the languages of vanilla JavaScript, CSS and HTML. Images are conveyed in SVG format to maintain resolution independence. Audio files must be in either MP3 or WAV format.

3 Specific Requirements

3.1 External Interface Requirements

The external requirement for the game is any PC or desktop browser. Once the user is on the game page the game will begin upon clicking the "Play" button and the 20 sec timer will begin, and the music will start playing. The program will then add a target into the 3 by 3 holes, which is the “Professor”. The User must use the mouse or touchscreen to click on the Professor and in order to accumulate points which are incremented in values of 10 points per hit.


3.2 User Interfaces

Start Screen: The game can only begin when User presses the “Play” button. Upon clicking “Play”, the game begins and the timer starts counting down. The User must use the mouse or touchscreen to press the Professor.

Game Over Screen: The game is over when timer reaches 0 at which the timer will stop and no Professor will appear on the screen. The music will also stop playing when the timer reaches 0. The final score is displayed and the Player will have the chance to play again.

3.1.2 Hardware Requirements

The game is intended to run in a web browser and should be able to run on most devices that support modern browsers to include desktops and laptops. In order to play the game, the user must have at least a mouse or touch screen, where applicable. In order to hear the music or on-hit sound effect, the Player must also have speakers, although that is optional.
